## Questions:- 

1. What is DevOps?
2. Why Learn DevOps?
3. What Are the Tools Used in DevOps?
4. What is SDLC?
5. What Problems Does DevOps Solve?


# Answers

1. what is Devops?
>>  DevOps is about delivering software faster, more reliably, and with better teamwork through automation and collaboration.

2. Why Learn DevOps? 
>> DevOps is a modern software development approach that combines development and operations to improve collaboration, automate processes, and deliver applications faster and more reliably. It emphasizes practices like continuous integration and continuous delivery (CI/CD), infrastructure as code, monitoring, and automation to reduce manual effort and errors. By breaking down silos between teams, DevOps enables organizations to release features quickly, maintain stability, and scale efficiently, making it a key driver of innovation and efficiency in today’s technology landscape.

3. What Are the Tools Used in DevOps?
>> Git/GitHub → Foundation for collaboration and version control.
   Jenkins/GitHub Actions → Automate CI/CD pipelines.
   Docker/Kubernetes → Standard for containerization and orchestration.
   Terraform/Ansible → Automate infrastructure provisioning and configuration.
   Prometheus/Grafana → Industry standard for monitoring and visualization.
   AWS/Azure/GCP → Backbone for scalable cloud infrastructure.

4. What is SDLC?
>> The Software Development Life Cycle (SDLC) is a structured process that guides how software is planned, developed, tested, deployed, and maintained to ensure quality, efficiency, and alignment with business goals. It provides a clear framework for teams to follow, reducing risks and improving collaboration

5. What Problems Does DevOps Solve?
>> DevOps solves the common challenges of traditional software development by breaking down silos between development and operations, automating repetitive tasks, and creating faster, more reliable delivery pipelines. It addresses problems like slow release cycles, error‑prone manual processes, poor collaboration, and lack of visibility into system health. With practices such as continuous integration, continuous delivery, infrastructure as code, and monitoring, DevOps ensures that software can be built, tested, deployed, and scaled efficiently. In essence, it transforms software delivery into a streamlined, collaborative, and automated process that reduces downtime, improves quality, and accelerates innovation.
